# Ручной чек-лист проверки в Autodesk Inventor

## 1) Подготовка
1. Откройте Inventor.
2. Откройте VBA Editor (`Alt+F11`).
3. Импортируйте модули из `src/` через `File -> Import File...`.
4. Выполните `Debug -> Compile VBAProject`.

Ожидаемо: компиляция проходит без ошибок синтаксиса VBA.

## 2) Первый запуск (новый чертеж)
1. Запустите макрос `RKM_CreateNewDrawingAndApplyFrameAndTitleBlock`.
2. Дождитесь создания нового Drawing document.

Проверить:
- на лист добавлена отдельная рамка;
- внизу справа добавлен отдельный штамп;
- штамп вставляется без runtime-ошибок VBA и отображает значения prompt (по умолчанию или пустые).

## 3) Проверка prompt-подстановки
1. Запустите макрос вставки штампа с массивом тестовых значений (15 строк).

Проверить:
- значения попали в ожидаемые ячейки штампа;
- порядок соответствует `docs/prompt-fields.md`;
- если передан некорректный массив, макрос использует безопасные значения и не падает.

## 4) Проверка обязательных статических текстов
Проверить наличие:
- в рамке слева: «Согласовано:», «Взамен инв. №», «Подпись и дата», «Инв. № подл.»;
- в штампе: «Заказчик:», «Стадия», «Лист», «Листов», заголовки ревизии, «Разработал», «A3».

## 5) Проверка удаления ГИП/Н.контроль
Убедиться визуально, что в итоговом штампе **нет** строк/подписей:
- «ГИП»
- «Н.контроль»

## 6) Повторный запуск на активном листе
1. Запустите `RKM_ApplyFrameAndTitleBlockToActiveSheet` повторно.

Проверить:
- старые экземпляры border/title block удаляются безопасно;
- вставляются новые;
- дубликаты на листе не накапливаются.

## 7) Пересборка ресурсов
1. Запустите `RKM_RebuildDefinitionsInActiveDrawing`.

Проверить:
- если старые definitions не использовались: они заменяются тем же именем;
- если definitions уже referenced: создаются версии `_v2`, `_v3`, ...;
- ошибок при пересборке нет.

## 8) Проверка режима редактирования
1. Попробуйте запустить любой макрос, находясь в активном режиме редактирования (если применимо).

Проверить:
- макрос корректно предупреждает о необходимости завершить edit-mode.
