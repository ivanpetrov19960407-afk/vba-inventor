# Ручной чек-лист: IDW-альбом + Excel + СПДС A3

## Предусловия
- Открыт Autodesk Inventor.
- Импортированы модули из `src/*.bas`.
- Выполнен `Debug -> Compile VBAProject`.
- Подготовлен Excel-файл по `docs/excel-format.md`.

## Сценарий A: альбом в активном чертеже
1. Откройте существующий `.idw`.
2. Запустите `Rkm_BuildAlbumFromExcel_OnActiveDrawing`.
3. В диалоге выберите Excel-файл.
4. Проверьте результат:
   - созданы/обновлены листы `ALB_*`;
   - на каждом листе ровно одна рамка и один штамп;
   - на каждом листе ровно 3 вида (front/top/side);
   - у всех 3 видов включены подписи (`ShowLabel=True`) и подписи не залезают на штамп;
   - виды не пересекают заблокированную область штампа.

## Сценарий B: альбом в новый `.idw` + SaveAs
1. Запустите `Rkm_BuildAlbumFromExcel_AndSaveAs`.
2. В диалоге выберите Excel-файл.
3. В диалоге сохранения выберите путь и имя `.idw`.
4. Проверьте, что файл сохранён и содержит листы `ALB_*`.

## Проверка ISO/ANSI
1. Смените активный стандарт документа (ISO first-angle / ANSI third-angle).
2. Повторно запустите сборку альбома.
3. Проверьте:
   - подпись бокового вида меняется: «Вид слева» (ISO) или «Вид справа» (ANSI);
   - геометрия видов остаётся в safeRect и не пересекает штамп.

## Идемпотентность
1. Повторно запустите тот же макрос с тем же Excel.
2. Убедитесь:
   - листы не дублируются;
   - рамка/штамп не дублируются;
   - штамп обновлён по новым данным (если Excel изменили).

## Self-test макрос
1. Откройте лист с моделью (или лист, где есть ссылочная модель).
2. Запустите `Rkm_SelfTest_Create3ViewsOnActiveSheet`.
3. Проверьте окно Immediate:
   - `views on active sheet >= 3`;
   - `blocked collisions = 0`.
4. Убедитесь, что `Debug.Assert` не остановил выполнение.

## Проверка диагностики
1. В Excel укажите несуществующий `MODEL_PATH`.
2. Запустите сборку.
3. Проверьте окно Immediate (`Debug.Print`): есть контекст `stage/sheet/model`.
4. При фатальной ошибке проверьте `MsgBox` формата:
   - `Album build failed (Err <number>): <description>`.
