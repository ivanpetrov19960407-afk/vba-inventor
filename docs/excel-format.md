# Формат Excel для сборки IDW-альбома

## Лист и заголовки
- Предпочтительное имя листа: `ALBUM`.
- Если `ALBUM` не найден, используется первый лист книги.
- Строка заголовков определяется автоматически (поиск `MODEL_PATH` в первых 10 строках).
- Начиная со следующей строки — данные по моделям.

## Режимы заголовков

### 1) Строгий режим (канонические заголовки)
- `MODEL_PATH`
- `CODE`
- `PROJECT_NAME`
- `DRAWING_NAME`
- `ORG_NAME`
- `STAGE`
- `SHEET`
- `SHEETS`

### 2) Tolerant режим (алиасы)
Поддерживаются алиасы (английские/русские варианты после нормализации), например:
- `Модель`, `Путь к модели`, `Model`, `Path` -> `MODEL_PATH`
- `№ док.`, `Номер документа` -> `CODE`
- `Наименование проекта`, `Наименование объекта` -> `PROJECT_NAME`
- `Наименование листа` -> `DRAWING_NAME`
- `Организация` -> `ORG_NAME`
- `Стадия` -> `STAGE`
- `Лист` -> `SHEET`
- `Листов` -> `SHEETS`

## Нормализация заголовков
Перед сопоставлением выполняется:
- `Trim`, `UCase`;
- замена `№ -> N`;
- очистка разделителей/повторных пробелов;
- транслитерация кириллицы в ASCII;
- замена пробелов на `_`.

Внутренние ключи в VBA всегда ASCII.

## Столбцы
### Обязательный
- `MODEL_PATH` — путь к `.ipt`.
  - Можно абсолютный путь.
  - Можно относительный путь (поиск: Workspace активного проекта -> папка Excel).

### Необязательные (prompt-поля штампа)
- `CODE`
- `PROJECT_NAME`
- `DRAWING_NAME`
- `ORG_NAME`
- `STAGE`
- `SHEET`
- `SHEETS`

Если поле пустое:
- для `SHEET` и `SHEETS` используется автозаполнение (номер листа и общее число листов);
- для остальных полей используются значения по умолчанию из `DefaultPromptMap()`.

## Соответствие колонок Excel и prompted-полей штампа
- `CODE` -> `<Prompt>CODE</Prompt>`
- `PROJECT_NAME` -> `<Prompt>PROJECT_NAME</Prompt>`
- `DRAWING_NAME` -> `<Prompt>DRAWING_NAME</Prompt>`
- `ORG_NAME` -> `<Prompt>ORG_NAME</Prompt>`
- `STAGE` -> `<Prompt>STAGE</Prompt>`
- `SHEET` -> `<Prompt>SHEET</Prompt>`
- `SHEETS` -> `<Prompt>SHEETS</Prompt>`
